# ConsultingLang Documentation

Welcome to the boardroom.

This folder contains the working documentation for ConsultingLang, a real joke language where programs are decks, runtime execution happens in the boardroom, and errors are misalignments.

## Start Here

- [Getting Started](getting-started.md): install, run, transpile, and audit decks.
- [Language Reference](language-reference.md): complete v1 syntax and semantics.
- [Collections](collections.md): arrays, map/filter/sort, objects, dictionaries, and maps.
- [CLI Reference](cli.md): command behavior and exit expectations.
- [Errors And Strictness](errors-and-strictness.md): misalignment messages and stricter-than-JavaScript rules.
- [Examples Guide](examples.md): what each bundled example demonstrates.

## Design Principles

- The joke is in the syntax, not in broken behavior.
- Programs that compile should run correctly.
- Error messages are a first-class feature.
- The language is stricter than JavaScript so ConsultingLang can provide better misalignment feedback.
- V1 is intentionally small: no module system, no type declarations, no async runtime, and no standard library beyond core deck operations.
